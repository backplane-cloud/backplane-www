# Investor Fact Sheet

## Mission

Build the best-in-class SaaS cloud engineering platform to accelerate cloud native transformations for businesses worldwide.

## Problems

- Cloud Cost - Enterprises are struggling to articulate and obtain value from their investment in public cloud platforms.

  - Little to no cost budget controls and enforcement at enterprise scale.

  - A culture of cost accountability and ownership is alien to engineering teams.

- Cloud Native Transformations are stalling.

  - Multi cloud governance is complex and requires a scalable approach.
  - No systematic way to track transformation efforts.
  - No centralised way to govern multi cloud landscapes.

  - Building Custom Internal Developer portals takes time.

## Opportunity

### FinOps

    - FinOps established itself borne form the DevOps moniker to introduce cloud cost practices into the DevOps workflow. Moving from On-Premises capital expenditure to Cloud Operational Expenditure moves responsibility to monitor the consumption-based pricing to the Product teams. Built upon the

      > "You build it, you run it philosophy"

- With over 5,000 companies, 16,000 members, a CAGR of 15.4% and market value in 2023 of $47.6M, this practice area is only going to continue to grow and be absorbed into Platform Engineering practices.

### Platform Engineering

- With `17,000 members` and a `CAGR of 25%` and `market value in 2023 of $5.1B`, again, like FinOps, this is the current state of practices around cloud that will grow as companies seek to streamline and maximise value from their cloud investments.

### Cloud Market

- With a `CAGR of 16.4%` and `market value in 2023 of $670B` and `estimated to be $1.4T by 2029`, it's these markets and combination of these practice areas that are going to underpin the trajectory of cloud. Whilst commoditisation will naturally occur, and AI and Quantum Compute yield more innovations, the fundamental governance and integration is where Backplane adds tremendous value through its Open Cloud Abstraction API (Backplane Core), and it's flagship Cloud Engineering Platform (Backplane Cloud). Whilst many tools operate in the build and orchestration space, fewer tools cohensively bring together the concerns of governance in a holistic way that compliments the existing tool landscape.

## Products

### Backplane Core

- This is an Open Source Cloud Abstraction API that provides multi cloud environment provisioning, and retrieval of Access, Cost and Policy data. It has been open sourced to help serve the platform engineering community with the rapid development and prototyping of internal developer portals.

  #### Benefits

  - Simplifies and speeds up building of Internal Development Portal
  - Consume Governance as a service into existing systems
  - Set Budgets at Org, Platform and Product levels with Approval Workflows
  - Engineer and Developer friendly with CLI and SDK.
  - Cross Cloud Access enforcement and compliance
  - Cross Cloud Policy enforcement and compliance
  - Cross Cloud Cost visibility and compliance
  - Service Integration e.g. Github repo provisioned for each Product

### Backplane Cloud

- A SaaS-based Cloud Engineering Platform (Modular platform for Cloud Governance, Delivery and Transformation).

  #### Benefits

        - All the benefits of Core but with a UI
        - Backplane Marketplace for Community extensions

### Backplane Modules

These are licensable extensions aim specifically at enterprises. Modules to be developed:

    #### Modules

    - Cloud Graph - provides capability to query all cloud resources across cloud platforms.

    - RBAC Proxy - provides the ability to manage RBAC assignmens across cloud platforms and provides visibility to inform permission posture compliance.

    - Policy Exemption Manager - aimed at Enterprise Architects to enforce architectural controls and allow streamlined exemption management for Platform and Product teams.

    - **Transformation Tracking** - augmenting the software catalog to represent on-prem/legacy workloads and score them for cloud native transformation.

## Model

- Backplane Software operates an Open Core model.

- With Backplane Core (Cloud Abstraction API) being fully open source.

- Backplane Cloud (SaaS Cloud Engineering Platform) is driven by subscription.

- Backplane Modules are licensable components that extend Backplane Cloud to deliver high-value and specificalised functionality aimed at enterprise customers.

## Strategy

> The commercial strategy is to problem value wihtin the Cloud engineering community and FinOps space to champion and steward Cloud Native Transformations. The Cloud Abstraction API delivers value to these communities upon which lays a foundation upon which engineers and step-up into the SaaS platform and module extensions to obtain more value.
